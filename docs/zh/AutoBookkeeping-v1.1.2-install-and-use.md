# 自动做账 v1.1.2 安装和使用说明

## 一、下载

GitHub Release:
https://github.com/Austin-C1/auto-bookkeeping/releases/tag/v1.1.2

第一次安装下载：

```text
AutoBookkeeping-Blank-v1.1.2.zip
```

已经安装旧版本，只更新程序下载：

```text
auto-bookkeeping-update-v1.1.2.zip
```

## 二、第一次安装

1. 先安装并打开 Docker Desktop。
2. 解压 `AutoBookkeeping-Blank-v1.1.2.zip`。
3. 双击 `一键安装启动.cmd`。
4. 程序会自动启动数据库、后端和页面。
5. 启动完成后会自动打开：

```text
http://127.0.0.1:18880/bookkeeping
```

v1.1.2 没有登录页、密码页、重置密码页，也不会再提示缺少认证。

## 三、端口

用户不需要手动改端口。

| 用途 | 地址 |
|---|---|
| 打开页面 | `http://127.0.0.1:18880/bookkeeping` |
| 后台接口 | `http://127.0.0.1:18001` |
| MySQL 数据库 | `127.0.0.1:23307` |

`127.0.0.1` 只代表当前电脑。别人要使用，需要下载并在自己的电脑上启动，不能直接打开你电脑上的链接。

## 四、旧版更新

旧版用户可以在“系统更新”页面检查更新，也可以下载 `auto-bookkeeping-update-v1.1.2.zip` 手动更新。

更新不会覆盖：

1. Crown 账号配置
2. WhatsApp 和 Telegram 群配置
3. 已生成的表格
4. 本机数据库
5. 本机配置文件

v1.1.2 启动时会检查后端版本。如果旧后端还占用端口，会自动重启到新版，避免新页面连接旧后端。

## 五、日常启动

安装成功后，日常使用下面任一方式启动：

1. 双击桌面快捷方式 `自动做账启动`。
2. 双击安装目录里的 `一键安装启动.cmd`。
3. 双击安装目录里的 `launch-blackcat.cmd`。

如果后台已经运行，只是浏览器页面关了，可以双击：

```text
open-blackcat-frontend.cmd
```

## 六、常见问题

### 打开后提示 Network Error

通常是 Docker Desktop 没打开、后端没有启动成功，或者手动改过端口。

处理方式：

1. 关闭浏览器页面。
2. 打开 Docker Desktop。
3. 重新双击 `自动做账启动`。
4. 不要手动改端口。

### 别人打不开我发的链接

这是正常现象。`127.0.0.1` 只代表自己的电脑。别人需要下载并在自己的电脑上启动。

### 还看到登录、密码、重置密码或缺少认证

说明打开的是旧版本、旧端口、旧浏览器标签页，或者旧后端还在占用端口。

处理方式：

1. 关闭旧页面。
2. 重新启动 v1.1.2。
3. 打开 `http://127.0.0.1:18880/bookkeeping`。
4. 如果仍然出现，退出旧的后端进程后再启动，或使用 v1.1.2 空白安装包重新安装。
