# 自动做账 v1.1.0 安装和使用说明

## 一、下载

从 GitHub Release 下载最新版：

https://github.com/Austin-C1/auto-bookkeeping/releases/tag/v1.1.0

第一次安装下载：

```text
AutoBookkeeping-Blank-v1.1.0.zip
```

已经安装过旧版，只更新程序下载：

```text
auto-bookkeeping-update-v1.1.0.zip
```

## 二、第一次安装

1. 先安装并打开 Docker Desktop。
2. 解压 `AutoBookkeeping-Blank-v1.1.0.zip`。
3. 双击 `一键安装启动.cmd`。
4. 程序会自动启动数据库、后端和前端。
5. 启动完成后会自动打开：

```text
http://127.0.0.1:18880/bookkeeping
```

本机版本默认免密进入，不需要输入密码，也不会再出现“重置密码”页面。

## 三、端口说明

用户不需要手动改端口。

| 用途 | 地址 |
|---|---|
| 打开页面 | `http://127.0.0.1:18880/bookkeeping` |
| 后台接口 | `http://127.0.0.1:18001` |
| MySQL 数据库 | `127.0.0.1:23307` |

`127.0.0.1` 只代表当前电脑。这个链接只能在安装程序的那台电脑上打开，不能发给别人直接访问。

如果别人要使用，需要让对方下载 GitHub 安装包，在对方自己的电脑上双击启动。

不要把页面端口手动改成 `80`。只改页面端口会导致前端找不到后端，点击“进入系统”时可能出现 `Network Error`。

## 四、日常启动

安装成功后，日常使用下面任一方式启动：

1. 双击桌面快捷方式 `自动做账启动`。
2. 双击安装目录里的 `一键安装启动.cmd`。
3. 双击安装目录里的 `launch-blackcat.cmd`。

如果后台已经在运行，只是浏览器页面关了，可以双击：

```text
open-blackcat-frontend.cmd
```

## 五、旧版更新

旧版用户可以在系统里的“系统更新”页面检查更新，也可以下载 `auto-bookkeeping-update-v1.1.0.zip`。

更新不会覆盖：

1. 本机登录状态。
2. 皇冠账号配置。
3. WhatsApp 和 Telegram 群配置。
4. 已生成的表格和本机数据库。

## 六、常见问题

### 打开后提示 Network Error

通常是后端没有启动成功，或手动改过端口。

处理方式：

1. 关闭浏览器页面。
2. 重新双击 `自动做账启动`。
3. 不要手动改端口。
4. 确认 Docker Desktop 已经打开。

### 别人点我发的链接打不开

这是正常现象。`127.0.0.1` 只代表自己电脑。别人需要下载并在自己的电脑上启动。

### 还看到重置密码页面

说明打开的是旧版本、旧端口或旧浏览器标签页。关闭旧页面后，重新启动 v1.1.0，并打开：

```text
http://127.0.0.1:18880/bookkeeping
```

v1.1.0 已删除旧的重置密码页面和后端接口。
