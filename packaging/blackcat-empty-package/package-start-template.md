# 自动做账功能使用说明书

适用版本：v{{VERSION}}

这份说明按日常做账顺序写。正常使用时，先完成账号和群配置，再按「赛前工作台」或「滚球工作台」生成表格。

## 一、启动程序

启动方式：

1. 双击桌面的 `自动做账启动`。
2. 或双击当前目录里的 `一键安装启动.cmd`。
3. 或双击当前目录里的 `launch-blackcat.cmd`。

启动完成后会自动打开：

```text
http://127.0.0.1:18880/bookkeeping
```

本机版本默认免密进入，不需要输入登录密码。

如果后台已经启动，只是页面被关掉，可以双击：

```text
open-blackcat-frontend.cmd
```

## 二、第一次启动会做什么

第一次启动会自动准备：

1. 本机 MySQL 数据库容器。
2. 程序后端。
3. 程序前端。
4. 桌面快捷方式 `自动做账启动`。

本安装包的数据库容器：

```text
{{CONTAINER_NAME}}
```

数据库端口：

```text
{{DATABASE_PORT}}
```

第一次启动会下载 `mysql:8.1`，网速慢时会等几分钟。

## 四、第一次使用前的配置

### 1. 配置皇冠账号

进入「皇冠账号管理」：

1. 填写账号名称。
2. 填写皇冠地址。
3. 填写用户名和密码。
4. 设置是否启用。
5. 保存账号。
6. 使用测试功能确认账号能读取。

### 2. 配置 WhatsApp 群

进入「WhatsApp 群配置」：

1. 点击刷新登录状态。
2. 如果未登录，用手机 WhatsApp 扫码。
3. 登录成功后，点击同步本机群聊。
4. 找到需要做账的群。
5. 给每个群设置角色。
6. 设置货币和汇率。
7. 保存。

常用群角色：

| 角色 | 说明 |
|---|---|
| 赛前上游 | 上游赛前确单群 |
| 赛前下游 | 下游赛前确单群 |
| 赛前公司 | 公司赛前跟单群 |
| 滚球上游 | 上游滚球确单群 |
| 滚球下游 | 下游滚球确单群 |
| 滚球公司 | 公司滚球跟单群 |
| 忽略 | 不参与做账 |

### 3. 配置 Telegram 群

进入「Telegram 群配置」：

1. 准备 Telegram 的 `API_ID` 和 `API_HASH`。
2. 在页面里填写并保存。
3. 用手机 Telegram 的「连接桌面设备」扫码。
4. 登录成功后，点击同步本机群聊。
5. 给群设置角色、货币和汇率。
6. 保存。

## 五、群消息识别和去重

程序会从已启用、已设置角色的群里读取确单。

识别重点：

1. 消息里需要有明确的比赛、联赛、盘口、赔率、金额和确认信息。
2. 含有「确认」的内容会优先作为确单处理。
3. 同一条注单被多次转发、重复提及或重复扫描时，程序会去重。
4. 赛前订单不需要投注时比分。
5. 滚球订单如果标题里带 `(0 - 0)` 这类比分，会写入「投注时比分」列。
6. 群里确认的赔率就是最终结算赔率，不再分退水前和退水后。

## 六、赛前表格生成流程

进入「赛前工作台」。

步骤：

1. 选择业务日期。
2. 设置扫描开始时间和结束时间。
3. 点击扫描 WhatsApp 赛前群。
4. 点击扫描 TG 赛前群。
5. 如需重新整理，使用重扫按钮。
6. 点击抓取赛果。
7. 点击一键生成。

也可以分别生成：

1. 赛前上游各群表格。
2. 赛前下游各群表格。
3. 盈亏水表格。
4. 公司盈亏表格。

推荐日常使用「一键生成」。

## 七、赛前表格内容说明

上游和下游群表格主要列：

1. 日期。
2. 序号。
3. 联赛类型。
4. 投注类型。
5. 比赛队伍。
6. 投注盘口及赔率。
7. 实际比分。
8. 投注额度。
9. 赛果。
10. 盈亏。

表格底部会统计：

1. 总投注额度。
2. 总盈亏。

盈亏水表格只统计有盈利结果的注单，主要是「赢」和「赢半」。

盈亏水计算规则：

```text
盈亏水 = (上游赔率 - 下游赔率) × 下游投注额
```

如果是赢半：

```text
盈亏水 = (上游赔率 - 下游赔率) × 下游投注额 × 0.5
```

如果上游赔率和下游赔率相同，不产生盈亏水。

公司投注额度计算规则：

```text
公司投注额度 = 下游合计投注额 - 上游投注额
```

当前版本公司投注赔率暂按上游赔率处理。

## 八、滚球表格生成流程

进入「滚球工作台」。

步骤：

1. 选择业务日期。
2. 设置扫描开始时间和结束时间。
3. 扫描 WhatsApp 滚球群。
4. 扫描 TG 滚球群。
5. 确认「投注时比分」已记录。
6. 点击抓取赛果。
7. 点击一键生成。

滚球消息标题里如果有比分，例如：

```text
足球 (滚球) 大 / 小 (0 - 0)
```

程序会把 `0 - 0` 写入「投注时比分」列。

滚球表主要列：

1. 日期。
2. 序号。
3. 联赛类型。
4. 投注类型。
5. 比赛队伍。
6. 投注盘口及赔率。
7. 投注时比分。
8. 实际比分。
9. 投注额度。
10. 赛果。
11. 盈亏。

## 九、盈亏计算说明

系统按亚洲盘结果计算。

| 赛果 | 含义 | 盈亏 |
|---|---|---|
| 赢 | 全赢 | 投注额 × 赔率 |
| 赢半 | 赢一半 | 投注额 × 赔率 × 0.5 |
| 走盘 | 退回本金 | 0 |
| 输半 | 输一半 | 投注额 × 0.5 的负数 |
| 输 | 全输 | 投注额的负数 |

示例：

```text
投注额 7500，赔率 0.9，结果赢
盈亏 = 7500 × 0.9 = 6750
```

```text
投注额 7500，赔率 0.9，结果赢半
盈亏 = 7500 × 0.9 × 0.5 = 3375
```

```text
投注额 7500，结果输
盈亏 = -7500
```

## 十、赛果抓取说明

赛果抓取用于填入实际结束比分。

使用时注意：

1. 先确认业务日期正确。
2. 队伍名称需要能和赛果源匹配。
3. 联赛名称越准确，匹配越稳定。
4. 比赛必须已经结束。
5. 抓取后检查「实际比分」列是否有空值。

没有实际比分时，程序不能正确产出赛果和盈亏。

## 十一、报表下载和查看

生成后的文件会出现在页面底部「文件生成结果」。

可以操作：

1. 查看文件类型。
2. 查看文件名。
3. 查看生成时间。
4. 点击下载。
5. 清除已生成文件。

Excel 打开时如果出现「受保护的视图」，点击启用编辑即可查看完整内容。

## 十二、系统更新

进入「系统更新」：

1. 点击检查更新。
2. 如果有新版本，查看版本号和更新说明。
3. 点击立即更新。
4. 等待更新完成。
5. 刷新页面或重新启动程序。

更新不会覆盖本机已有配置：

1. 登录状态。
2. 皇冠账号。
3. WhatsApp 群配置。
4. Telegram 群配置。
5. 已生成文件。
6. Docker 数据库。
7. 本机配置。

## 十三、常见问题

### 1. 扫描不到群消息

检查：

1. 群是否已同步。
2. 群角色是否设置正确。
3. 群是否启用。
4. 日期和时间范围是否覆盖消息。
5. 消息里是否有确认内容。
6. WhatsApp 或 Telegram 是否登录成功。

### 2. 生成表格后没有盈亏

检查：

1. 实际比分是否已经填入。
2. 盘口是否能识别。
3. 投注类型是否是大小或让球。
4. 金额是否识别成功。
5. 赛果是否为空。

### 3. 盈亏水为空

盈亏水为空通常表示：

1. 没有盈利注单。
2. 上游和下游赔率相同。
3. 没有匹配到同一订单的下游拆单。
4. 下游投注金额为空。

### 4. 滚球没有投注时比分

检查原消息标题是否包含类似：

```text
(0 - 0)
```

没有这个格式时，程序无法自动知道投注时比分。

### 5. 桌面快捷方式打不开

处理方式：

1. 双击当前目录里的 `launch-blackcat.cmd`。
2. 程序会自动修正桌面的 `自动做账启动`。
3. 如果仍打不开，删除旧的 `自动做账启动`，再双击 `launch-blackcat.cmd`。
